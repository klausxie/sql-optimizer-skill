# 详细设计 4：数据契约版本管理

## 1. 设计目标

确保数据契约的版本兼容，支持运行时版本检测和数据迁移。

## 2. 版本元数据设计

### 2.1 Schema 元数据字段

```json
{
  "x-version": {
    "major": 1,
    "minor": 2,
    "patch": 0,
    "released": "2024-03-17"
  },
  "x-compatibility": {
    "forward": ["1.0.x", "1.1.x"],
    "backward": ["1.1.x", "1.2.x"]
  }
}
```

### 2.2 字段级元数据

```json
{
  "properties": {
    "baseline_metrics": {
      "type": "object",
      "x-addedIn": "1.2.0",
      "x-default": {}
    }
  }
}
```

### 2.3 设计思路

**x-version**：声明 Schema 版本

**x-addedIn**：标记字段在哪个版本添加，帮助旧版本代码理解新字段

**x-default**：新字段的默认值，旧版本读取时可自动填充

**x-compatibility**：声明兼容范围，帮助运行时判断是否兼容

## 3. 版本检测机制

### 3.1 运行时检测

- 加载数据时提取版本信息
- 与当前契约版本对比
- 不兼容时给出警告

### 3.2 迁移策略

- 提供迁移脚本支持旧版本升级
- 渐进式迁移，不强制用户升级

## 4. 最佳实践

1. 尽量不破坏兼容
2. 新增字段使用 x-addedIn 标记
3. 运行时自动检测版本兼容性
4. Schema 变更必须更新文档
