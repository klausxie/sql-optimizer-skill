# 详细设计 7：Web 管理界面设计

## 1. 设计目标

提供可视化的管理界面，提升用户体验。

## 2. 功能模块

| 模块 | 功能 |
|------|------|
| 仪表盘 | 统计概览、流水线状态、最近运行 |
| 运行管理 | 运行列表、详情查看、控制操作 |
| 数据查询 | JSONPath 查询、版本对比、契约验证 |
| 报告 | 优化建议、风险报告、趋势分析 |
| 配置 | 项目配置管理 |
| 契约 | 数据契约管理 |

## 3. 界面设计

### 3.1 风格

- 深色主题 + 青色/紫色强调色
- JetBrains Mono 代码字体
- 卡片式布局 + 悬停动效

### 3.2 布局

```
┌────────────────────────────────────────┐
│  ⚡ SQL Optimizer                      │
├──────────┬───────────────────────────┤
│  导航     │  主内容区                   │
│           │                           │
│  - 仪表盘 │  统计卡片                  │
│  - 运行   │  流水线状态                │
│  - 数据   │  运行列表                  │
│  - 报告  │                           │
│  - 配置   │                           │
│  - 契约   │                           │
└──────────┴───────────────────────────┘
```

## 4. 设计思路

### 4.1 底层调用

界面底层调用两个 CLI 完成实际操作：
- sqlopt-cli：运行管理
- sqlopt-data：数据查询

### 4.2 交互流程

1. 用户在界面操作
2. 界面调用 CLI
3. CLI 执行并返回结果
4. 界面展示结果

### 4.3 技术选型

- React + TypeScript
- 自定义组件（参考设计风格指南）
- Zustand 状态管理
