## 二、核心设计决策

| 决策项 | 结论 |
|--------|------|
| **数据格式** | 继续使用 JSON（保持现状） |
| **版本管理** | Git + JSON 扩展混合方案 |
| **CLI 设计** | 双 CLI：sqlopt-cli（优化）+ sqlopt-data（数据） |
| **分层存储** | 项目级缓存 + 运行级产物分离 |
| **Java 分析** | 短期不实施（后期方向） |

## 三、7 阶段流程

| 阶段 | 名称 | 功能 | 耗时节点 |
|------|------|------|----------|
| 1 | Discovery | 连接数据库、采集表结构、解析XML | DB 🐢 |
| 2 | Branching | 分支展开（3种策略） | - |
| 3 | Pruning | 静态分析、风险标记、聚合剪枝 | CPU ⚡ |
| 4 | Baseline | EXPLAIN、执行SQL、采集性能 | DB 🐢 |
| 5 | Optimize | 规则引擎 + LLM优化 | LLM ⏱️ |
| 6 | Validate | 语义验证，性能对比、结果集验证 | DB 🐢 |
| 7 | Patch | 生成补丁、用户确认，应用 | FS 💾 |

## 五、数据目录设计

### 5.1 分层结构

```
项目根目录/
│
├── .sqlopt/                        # 项目级持久存储
│   ├── cache/
│   │   ├── db_schemas/          # 表结构缓存（按DB哈希）
│   │   └── sqlmap_cache/        # SQL模板缓存
│   └── history/                   # 历史运行归档
│
├── runs/                         # 运行时产物
│   └── {run_id}/
│       ├── project_context/       # 全局上下文
│       ├── sqlmap_catalog/      # SQL目录
│       └── supervisor/          # 运行状态
│
└── contracts/                    # 数据契约 Schema
```

### 5.2 设计思路

**分层原则**：
- 项目级缓存：可被多次运行复用，减少重复采集
- 运行级产物：本次运行的完整数据，独立隔离

**缓存策略**：
- 表结构缓存：按数据库连接哈希存储，支持 TTL 过期
- SQL 模板缓存：按 XML 文件 hash 缓存，文件变化自动失效

**加载策略**：
- 索引快速加载：仅加载 index.json 即可列出所有 SQL
- 详情按需加载：需要查看详情时再加载具体文件
- 阶段批量加载：按阶段目录批量加载进行全量分析

## 六、版本管理

### 6.1 契约版本管理

```
contracts/
├── schemas/                    # 主 Schema（最新版本）
├── snapshots/                  # 历史版本快照
│   ├── v1.0.0/
│   ├── v1.1.0/
│   └── v1.2.0/
├── versions.json              # 版本索引
└── CHANGELOG.md             # 变更日志
```

### 6.2 设计思路

**混合方案**：
- schemas/ 目录始终保持最新版本，开发时直接修改
- snapshots/ 存放历史版本快照，用于版本回溯
- versions.json 作为版本索引，记录各版本的契约版本映射
- CHANGELOG.md 记录详细的变更内容

**语义化版本**：
- MAJOR：不兼容的结构变更
- MINOR：新增可选字段，向后兼容
- PATCH：文档格式修正

## 七、数据契约版本管理

### 7.1 版本元数据设计

每个 Schema 文件包含版本元数据：
- x-version：版本号和发布日期
- x-addedIn：字段添加版本
- x-default：默认值
- x-compatibility：向前/向后兼容声明

### 7.2 设计思路

**版本检测**：
- 运行时自动检测数据版本与契约版本的兼容性
- 不兼容时给出明确警告

**渐进迁移**：
- 新增字段使用 x-addedIn 标记为可选
- 提供迁移脚本支持旧版本数据升级

### 8.1 格式对比

| 格式 | 人类可读 | 体积 | 工具支持 | 适用场景 |
|------|----------|------|----------|----------|

| TOML | ★★★★★ | 小 | ★★★☆☆ | 简单配置 |
| MessagePack | ★☆☆☆☆ | 小 | ★★★☆☆ | 紧凑传输 |

### 8.2 设计思路

**当前选择：JSON**

- 后续可渐进改进：配置文件改用 YAML
