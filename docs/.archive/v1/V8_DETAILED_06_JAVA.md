# 详细设计 6：Java 代码分析扩展

## 1. 愿景

当前 SQL Optimizer 主要分析 MyBatis XML，未来可扩展到 Java 源代码分析。

## 2. 功能设计

### 2.1 Mapper 接口映射

检测 Java 方法与 MyBatis XML 中 SQL 的映射关系：
- 检测 @Select/@Insert/@Update/@Delete 注解
- 查找 mapper.method() 调用
- 建立完整的调用链

### 2.2 N+1 查询检测

检测在循环中执行数据库查询的问题：
- 解析循环结构
- 检测循环内的 mapper 调用
- 生成风险报告

### 2.3 缺失 LIMIT 检测

检测可能返回大量数据的查询：
- 检测无 LIMIT 子句的 SELECT
- 结合表数据量评估风险

### 2.4 全表扫描检测

检测可能导致全表扫描的模式：
- LIKE 前导通配符
- NOT IN
- OR 条件
- IS NULL
- 函数作用于列

## 3. 设计思路

### 3.1 技术选型

使用 javalang 库解析 Java AST

### 3.2 架构设计

作为独立模块：
- 默认关闭（性能考虑）
- 可配置启用
- 静态分析，有局限性

### 3.3 实施路线

- Phase 1：基础能力（解析器、注解检测）
- Phase 2：模式检测（N+1、LIMIT、全表扫描）
- Phase 3：深度分析（数据流、调用链）
- Phase 4：可视化（报告、IDE 集成）

## 4. 注意

- 短期不实施，作为后期方向
- 静态分析有局限性，可能误报
- 需要考虑性能影响
