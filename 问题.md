## 问题清单

### 用户反馈的问题

| # | 问题 | 精准描述 |
|---|------|----------|
| 1 | **找的太慢** | SKILL.md 里写的 `bin/sqlopt-cli.cmd` 不存在，实际路径是 `runtime/scripts/sqlopt_cli.py`。文档与代码不同步，导致多次试错。 |
| 2 | **缺少数据量维度** | 优化建议（`SELECT_STAR`, `NO_LIMIT`）是规则模板硬编码的，未结合 `tableStats.estimatedRows: 215`。215 行的小表全表扫描根本不是问题，建议价值低。 |
| 3 | **下一步建议变成 CLI 命令** | 报告输出 `PYTHONPATH=python python3 scripts/sqlopt_cli.py resume...`，但用户在 OpenCode 环境里，应该说"继续执行"而不是让用户复制命令。 |
| 4 | **validate-config 作为独立命令多余** | 用户几乎不需要单独验证配置，应该是 `run` 的 preflight 自动检查，失败才暴露。 |
| 5 | **脚本输出乱码** | Windows 下 `\u25b6` 等 Unicode 字符显示为乱码，应该检测环境后降级为 ASCII 或设置 `chcp 65001`。 |
| 6 | **未经同意自行修改数据库** | agent 在未授权情况下执行了 `ALTER TABLE ADD COLUMN`，这是 agent 行为问题，但 skill 设计上也没有明确"只读验证"边界。 |
| 7 | **命令命名冗余** | 当前只有一个 `sql-optimizer` skill。用户可能记混，或之前存在过 `/sql-optimize` 别名。建议统一为 `sql-opt`。 |

| # | 问题 | 精准描述 |

| 8 | **语义错误信息深埋** | `(1054, "Unknown column 'type'")` 藏在 JSON 中间，`blockers.summary.json` 只显示 `VALIDATE_SEMANTIC_ERROR`。顶层报告应该直接说"缺失列: users.type"。 |

## 优先级与修复方向

| 优先级 | 问题 | 修复方向 |
|--------|------|----------|
| **P0** | 下一步建议是 CLI | 输出结构化指令供 agent 执行，而非让用户复制命令 |
| **P0** | 语义错误不直观 | 顶层报告直接展示缺失列/表，而非 generic error code |
| **P1** | CLI 路径错误 | 修正 SKILL.md 或创建 `bin/sqlopt-cli.cmd` 包装脚本 |
| **P1** | 输出乱码 | 检测 Windows 后用英文输出，或设置 UTF-8 编码 |
| **P2** | 缺少数据量建议 | 根据 `estimatedRows` 调整规则严重级别（小表降低 FULL_SCAN 风险等级） |
| **P2** | run/resume 混淆 | 明确区分或合并语义，文档说明 `run` 会从头开始 |

| **P3** | 命令命名 | 统一为 `sql-opt` 或明确区分 `/sql-optimize` vs `/sql-optimizer` |

## 具体改进建议

### 1. CLI 路径问题

**现状**: SKILL.md 写的是
```
bin/sqlopt-cli.cmd
```

**实际路径**:
```
runtime/scripts/sqlopt_cli.py
```

**建议**:
- 方案 A: 创建 `bin/sqlopt-cli.cmd` 作为包装脚本
- 方案 B: 更新 SKILL.md 中的路径

```cmd
@echo off
python "%~dp0..\runtime\scripts\sqlopt_cli.py" %*
```

---

### 2. 输出乱码问题

**现状**: Windows 下 Unicode 字符显示为乱码

```
\u25b6 开始阶段...
\u2713 完成阶段...
```

**建议**:

```python
import sys
import platform

def safe_print(text: str):
    """检测环境后选择输出方式"""
    if platform.system() == 'Windows' and sys.stdout.encoding != 'utf-8':
        # 降级为 ASCII
        text = text.replace('\u25b6', '>').replace('\u2713', '[OK]').replace('\u2139', '[i]')
    print(text)
```

### 3. 语义错误信息展示

**现状**: `blockers.summary.json` 只显示

```json
{
  "code": "VALIDATE_SEMANTIC_ERROR",
  "severity": "degradable"

```

**建议**: 增加结构化错误详情

```json
  },
  "human_message": "SQL 引用了不存在的列: users.type"
}
```

### 4. 数据量感知的优化建议

**现状**: 固定规则

```
FULL_SCAN_RISK: warn  # 无论表大小都触发
```

**建议**: 根据 `tableStats.estimatedRows` 调整

```python
def evaluate_full_scan_risk(table_stats, rule_config):
    rows = table_stats.get('estimatedRows', 0)
    
    if rows < 1000:
        return None  # 小表不需要索引，跳过
    elif rows < 10000:
        return {'severity': 'info', 'message': f'表约 {rows} 行，可考虑添加索引'}
    else:
        return {'severity': 'warn', 'message': f'表约 {rows} 行，建议添加索引避免全表扫描'}
```
